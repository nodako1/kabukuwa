# カブクワ 次期追加機能・修正仕様書

## とっておきの虫かご・個体コレクション Ver.5

- 作成日: 2026-07-21
- 対象: `nodako1/kabukuwa`
- 対象環境: スマートフォン向けWebアプリ（Vercel本番環境）
- 前提: 「自分で仕掛けるバナナトラップと翌朝の再訪 Ver.4」が実装済み
- 対象セーブ: Schema Version 5からVersion 6への移行
- ステータス: Ver.4完了後に着手可能

## 1. 結論

次の追加機能では、捕まえた虫を種類単位の集計だけでなく、1匹ずつ見返せる「とっておきの虫かご」を追加する。

現在も各個体には、種類、大きさ、捕獲日、時刻、場所、木、調査ポイント、捕獲元、ランキング対象状態が保存されている。しかし図鑑では「何匹捕まえたか」と「最大サイズ」だけへ集約されるため、捕獲結果を閉じると、最大個体以外の思い出を見返しにくい。

Ver.4では、自分で仕掛けたトラップによって同じ種類を複数捕まえる機会が増える。次段階では出現率や報酬を増やさず、捕まえた1匹ごとの記録を価値へ変える。お気に入り登録は完全に任意とし、報酬、能力、連続記録、未登録ペナルティへ接続しない。

## 2. この段階で追加する理由

Ver.4までの報酬は、主に捕獲直後の詳細と種類別図鑑へ残る。

~~~text
木や仕掛けを覗く
  ↓
虫影を見つける
  ↓
捕まえて大きさを見る
  ↓
種類別の捕獲数と最大サイズへ集約される
~~~

Ver.5では次の振り返りを追加する。

~~~text
捕まえた個体を1匹ずつ保存
  ↓
種類ごとの個体一覧で見比べる
  ↓
場所・時間・見つけ方を思い出す
  ↓
好きな個体を「とっておき」にする
  ↓
お気に入りだけを虫かごで眺める
~~~

これにより、小さな個体や同種の再捕獲も「外れ」ではなく、夏休みの思い出になる。通常の探索と宝探しを主役のまま維持し、既存データを活かして低い実装リスクで捕獲後の満足度を上げられる。

## 3. 体験目標

- 捕まえた1匹ごとに、いつ、どこで、どう見つけたかを思い出せる。
- 最大個体でなくても、好きな虫を「とっておき」として残せる。
- 同じ種類の大きさや捕獲場所を、迷わず見比べられる。
- 通常の木、固定トラップ、自分で仕掛けたトラップの違いを記録として楽しめる。
- お気に入り登録をしなくても、探索、図鑑、観察ノートを従来どおり楽しめる。
- 未発見種の正確な場所や出現条件を漏らさず、宝探しを維持する。
- 子どもや家族が使う表現として「標本」「選別」ではなく「とっておきの虫かご」を使う。

## 4. 対象範囲

### 4.1 Ver.5で実装するもの

1. 発見済み種類から開く個体一覧
2. 個体ごとの詳細記録
3. 「新しく捕まえた順」と「大きい順」の並び替え
4. 30件単位の追加表示
5. 捕獲結果と個体詳細からのお気に入り設定・解除
6. 図鑑内の「とっておき」タブ
7. 図鑑内の3段階遷移と戻る操作
8. 自己ベスト、または自己ベストとの差の表示
9. 捕獲元とランキング対象状態の説明
10. セーブSchema Version 6
11. 不正なお気に入り参照の自動修復

### 4.2 Ver.5で実装しないもの

- 捕獲個体の削除、逃がす操作
- 個体名、ニックネーム、自由入力メモ
- お気に入り数の上限
- お気に入り報酬、能力、称号、デイリーテーマ
- 捕獲個体の交換、共有、画像書き出し
- オンラインランキング
- クラウド保存、アカウント同期
- 新しい虫、出現場所、出現率、サイズ範囲の変更
- 2匹を選択して横並びにする高度な比較画面
- 本番イラスト、BGM、環境音
- 虫かごの配置換えや飼育シミュレーション

## 5. 用語と表示方針

| 内部用語 | 画面表示 | 方針 |
| --- | --- | --- |
| `Specimen` | 捕まえた虫／個体 | 一般画面で「標本」と呼ばない |
| favorite | とっておき | 星印と文章を併用する |
| collection | 虫図鑑 | 既存名称を維持する |
| favorite list | とっておきの虫かご | 好きな個体だけを見る場所 |
| ranking eligible | 将来のランキング対象 | 広告公平性の既存説明を維持する |

大きさだけが価値だと感じさせないよう、初期並び順は「新しく捕まえた順」とする。「最大」「自己ベスト」は目立たせるが、お気に入りの設定条件にはしない。

## 6. 基本UXフロー

### 6.1 捕獲直後

1. 従来どおり虫影をタップする。
2. 捕獲演出後、種類、大きさ、場所、時刻などの詳細を表示する。
3. 従来の主操作「わかった」を維持する。
4. 任意の副操作として「☆ とっておきにする」を表示する。
5. 登録後は「★ とっておきにした」に変わる。
6. 登録してもしなくても「わかった」で元の接写へ戻れる。

お気に入り操作でゲーム内時間、観察進捗、捕獲結果、ランキング対象状態を変更しない。

### 6.2 種類から個体を見る

1. 図鑑の「虫図鑑」タブを開く。
2. 発見済みの種類カードをタップする。
3. その種類の個体一覧へ切り替わる。
4. 「新しい順／大きい順」を切り替える。
5. 個体カードをタップして詳細を開く。
6. 詳細からお気に入りを変更する。
7. 戻ると同じ種類一覧、さらに戻ると種類一覧へ戻る。

### 6.3 とっておきだけを見る

1. 図鑑の「とっておき」タブを開く。
2. 種類をまたいで、お気に入り個体を新しい順に表示する。
3. 個体カードから詳細を開く。
4. お気に入りを解除すると、詳細を閉じた時点で一覧から外れる。
5. 0匹の場合は、捕獲後や個体詳細から任意で登録できることを説明する。

## 7. 図鑑内の画面階層

モーダルを重ねず、既存の図鑑シート1枚の中で表示を切り替える。

既存の2タブを「虫図鑑」「とっておき」「観察ノート」の3タブへ拡張する。左右矢印キーは3タブを循環し、360px幅でも横スクロールを発生させない。

~~~ts
type EncyclopediaTab = "insects" | "favorites" | "journal";

type CollectionPage =
  | { type: "index" }
  | { type: "species"; insectId: InsectId }
  | { type: "specimen"; specimenId: string; returnTo: "species" | "favorites" };
~~~

`CollectionPage`と並び順、表示件数はUIローカル状態とし、セーブしない。

~~~mermaid
stateDiagram-v2
    [*] --> SpeciesList: 虫図鑑
    SpeciesList --> SpecimenList: 発見済み種を選ぶ
    SpecimenList --> SpecimenDetail: 個体を選ぶ
    SpecimenDetail --> SpecimenList: 戻る
    SpecimenList --> SpeciesList: 戻る
    SpeciesList --> Favorites: とっておき
    Favorites --> SpecimenDetail: 個体を選ぶ
    SpecimenDetail --> Favorites: 戻る
~~~

### 7.1 戻る操作

- 画面内の「戻る」ボタン、Escape、ブラウザ戻るを同じ階層移動として扱う。
- `SpecimenDetail`では直前の個体一覧へ戻る。
- `SpecimenList`では種類一覧へ戻る。
- ルート階層で戻ると図鑑シートを閉じる。
- 戻った後は、直前に選んだ種類カードまたは個体カードへフォーカスを戻す。
- タブ切り替え時は階層をルートへ戻し、別タブの詳細状態を背後に残さない。

## 8. 種類別の個体一覧

### 8.1 カード表示

各個体カードへ次を表示する。

- 大きさ（0.1mm単位）
- 捕獲日と時刻
- 捕獲場所
- とっておきの星
- 自己ベストの場合は「自己ベスト」
- それ以外は必要に応じて「ベストまで -X.Xmm」

木、調査ポイント、捕獲元、ランキング状態は詳細画面で表示し、一覧を情報過多にしない。

### 8.2 並び順

| 並び順 | 基準 | 同値時の順序 |
| --- | --- | --- |
| 新しい順 | `day`降順、`caughtAtMinutes`降順 | `id`昇順で安定化 |
| 大きい順 | `sizeMm`降順 | 新しい順、最後に`id`昇順 |

- 初期値は「新しい順」とする。
- 種類ごとに最後の並び順を保存しない。図鑑を開き直すと新しい順へ戻る。
- 並び替えは表示状態であり、セーブ対象にしない。

### 8.3 件数が多い場合

- 初期表示は30件とする。
- 「さらに30匹見る」で次の30件を追加する。
- 並び順を変えた場合は表示件数を30件へ戻す。
- 30件以下では追加ボタンを表示しない。
- 仮想スクロールは初期実装で必須にせず、500個体で性能基準を満たさない場合に導入する。

## 9. 個体詳細

個体詳細へ次を表示する。

1. 虫の種類
2. 大きさ
3. 自己ベスト、またはベストとの差
4. 捕獲日
5. 捕獲時刻
6. 場所
7. 木の名前
8. 見つけた調査ポイント
9. 捕獲元
10. ランキング対象状態
11. とっておき設定

### 9.1 捕獲元の表示

| `captureSource` | 表示 |
| --- | --- |
| `tree` | 木を調べて発見 |
| `fixed-banana` | 裏庭のバナナトラップ |
| `fixed-light` | 裏庭のライトトラップ |
| `player-banana` | 自分で仕掛けたバナナトラップ |

捕獲元の表示ロジックは、捕獲結果と個体詳細で同じヘルパーを使用する。

プレイヤートラップ個体は`captureSource === "player-banana"`、`spotId === trap.id`、`treeId === 設置木ID`、`inspectionPointId === "${tree.id}:player-banana-trap"`として保存される。最後の値は`TreeDefinition.inspectionPoints`に存在しない合成IDなので、通常の調査ポイント検索へ渡さず「自分で仕掛けたバナナトラップ」と表示する。

トラップ回収後は`playerTrapKit.activeTrap`が削除される。個体詳細はアクティブトラップへ依存せず、保存済み`Specimen`と`treeById`だけから場所と木を復元する。

### 9.2 旧データや不足参照

- 木や調査ポイントが現在のコンテンツから削除されていても、個体を消さない。
- 場所が分かる場合は場所を表示する。
- 木が分からない場合は「村の木のそば」と表示する。
- 調査ポイントが分からない場合は「見つけた場所の詳しい記録はありません」と表示する。
- 未知の参照を理由にセーブ全体を破棄しない。

## 10. とっておき設定

### 10.1 コマンド

~~~ts
type GameCommand =
  | ExistingGameCommand
  | {
      type: "SET_SPECIMEN_FAVORITE";
      specimenId: string;
      favorite: boolean;
    };
~~~

`TOGGLE`は使用しない。二重タップ、コマンド再送、再読込後の再操作で意図せず反転しないよう、最終状態を明示する。

### 10.2 ルール

- 存在する`Specimen.id`だけを登録できる。
- 既に同じ状態なら何も変更しない。
- お気に入り数に上限を設けない。
- 設定・解除でゲーム内時間を進めない。
- `revision`は実際に状態が変わった場合だけ増やす。
- 観察テーマ、日記、捕獲数、最大サイズ、ランキングへ影響しない。
- 未発見種をお気に入り操作から作成できない。
- 新規登録は`favoriteSpecimenIds`の末尾へ追加し、解除時は対象IDだけを除く。
- 一度解除した個体を再登録した場合は末尾へ追加し直す。
- とっておき一覧は配列を逆順にして「最近とっておきにした順」で表示し、追加の登録時刻は持たない。

## 11. データ設計

### 11.1 GameState

~~~ts
interface GameState {
  schemaVersion: 6;
  contentVersion: 6;
  // 既存項目
  favoriteSpecimenIds: string[];
}
~~~

`Specimen`自体へ`favorite`を持たせず、ID集合として分離する。これにより、既存個体データを更新せず、一覧や詳細から同じ状態を参照できる。

### 11.2 状態不変条件

- `favoriteSpecimenIds`に重複がない。
- すべてのIDが`specimens`内の実在個体を参照する。
- `specimens`の並び順をお気に入り操作で変更しない。
- 同じ`Specimen.id`が複数存在するセーブは不正として扱う。
- `Specimen`本体と`pendingOutcome`内の個体コピーへお気に入り状態を埋め込まない。
- お気に入り操作で`Specimen`本体と`pendingOutcome`を変更しない。

## 12. セーブSchema Version 6

### 12.1 Version 5からの移行

- 既存の全ゲーム状態を保持する。
- 全個体、日次プラン、観察ノート、プレイヤートラップ状態を保持する。
- `favoriteSpecimenIds: []`を追加する。
- 現在開いている通常接写、プレイヤートラップ接写、捕獲結果を保持する。
- Version 5原本を`kabukuwa.save.pre-v6`へバックアップしてから保存する。
- 既存の`kabukuwa.save.pre-v5`は上書きせず、Version 4以前へ戻せる原本を維持する。
- 過去個体をサイズなどから自動でお気に入りにしない。

### 12.2 修復

保存されたお気に入りIDが存在しない場合:

1. 不正な参照だけを一覧から除く。
2. 正常な個体、お気に入り、他のゲーム状態を保持する。
3. 重複IDは最初の1件へ正規化する。
4. 修復だけで捕獲個体を削除しない。
5. 修復通知は通常プレイを妨げない短い案内にする。

構造検証後、厳格な意味検証より前に参照修復を行う。修復可能な参照1件のために最新セーブ全体をバックアップへ巻き戻さない。

## 13. 既存機能との統合

### 13.1 虫図鑑

- 未発見カードは従来どおりヒントのみ表示する。
- 未発見カードは個体一覧を開けない。
- 発見済みカードへ捕獲数、最大サイズ、とっておき数を表示する。

### 13.2 捕獲結果

- 通常の木、固定トラップ、プレイヤートラップのすべてで同じお気に入り操作を表示する。
- お気に入り操作後も捕獲結果シートを開いたままにする。
- 捕獲演出中はお気に入り操作を表示しない。
- `pendingOutcome`中、通常接写中、プレイヤートラップ接写中の3つのReducer操作ガードすべてで`SET_SPECIMEN_FAVORITE`を許可する。
- 設定・解除で接写、`pendingOutcome`、`pendingBoundaryEvent`を変更しない。
- 17:55の捕獲結果で設定しても、迎えは結果を閉じるまで保留する。

### 13.3 観察ノート

- 日記内容、捕獲数、最大個体の決定方法を変更しない。
- 日記の個体リンクはVer.5の対象外とする。
- お気に入り設定を観察行動として記録しない。

### 13.4 プレイヤートラップ

- `captureSource === "player-banana"`の個体も通常どおり一覧と詳細へ表示する。
- トラップ回収後も個体詳細の木、場所、捕獲元を見返せる。
- Ready結果、存在判定、種類、サイズへお気に入り状態を入力しない。

## 14. 広告とランキングの公平性

- お気に入り設定は出現率、種類、サイズ、ランキング対象状態を変えない。
- `rankingEligible`を表示時に再計算せず、個体へ保存された値を使う。
- ランキング対象外個体もお気に入りにできる。
- 対象外個体には「図鑑・採集記録のみ」を表示する。
- お気に入り数やお気に入り率をランキングへ使わない。
- 広告視聴をお気に入り数、一覧件数、詳細閲覧の条件にしない。

## 15. UI・アクセシビリティ

- 全ボタンと個体カードのタップ領域を44×44 CSS px以上にする。
- 星だけで状態を示さず、「とっておきにする／とっておきから外す」を読み上げる。
- `aria-pressed`でお気に入り状態を公開する。
- 種類一覧、個体一覧、詳細の見出し階層を保つ。
- 並び替えはラベル付きの2択操作とし、選択中を色だけで示さない。
- 個体詳細を開いたとき、詳細見出しへフォーカスを移す。
- 戻ったとき、元のカードへフォーカスを戻す。
- ブラウザ戻る、Escape、画面内戻るを同じ階層動作にする。
- 360×640で、種類名、大きさ、場所名、お気に入り操作が横にはみ出さない。
- 200%拡大時も主操作が見え、横スクロールを要求しない。
- `prefers-reduced-motion`では星の登録演出を停止する。
- スクリーンリーダーへ未発見種の場所や個体数を漏らさない。

## 16. エラー・境界ケース

- 存在しない個体IDのコマンドは状態を変更しない。
- 同じ`favorite: true`を二重送信しても1件だけ保持する。
- 同じ`favorite: false`を二重送信しても他の個体を変更しない。
- 捕獲直後にお気に入りを設定して再読み込みしても維持する。
- お気に入り詳細で解除後、戻った一覧に対象を残さない。
- 種類別一覧で解除しても、その個体自体は一覧に残す。
- 0匹、1匹、30匹、31匹、500匹で表示を確認する。
- 同じ大きさ、同じ日、同じ時刻でも`id`で安定した順序にする。
- コンテンツ更新で木や調査ポイントが消えても個体詳細を開ける。
- 日送り、18時の迎え、20時の日記、接写保留中もお気に入りデータを失わない。
- 接写中に図鑑は開けない既存のモーダル排他を維持する。
- `RESET_GAME`だけがお気に入りを含む全個体記録を初期化する。

## 17. 性能要件

- 500個体保存時に図鑑を開いても、明確な操作停止を発生させない。
- 種類別の抽出、自己ベスト、並び替えは`useMemo`等で表示単位に計算する。
- 1カードごとに全個体を再走査しない。
- 初期描画は最大30個体とする。
- お気に入り設定時にフィールドや接写の大型描画を再生成しない。
- LocalStorageにはID配列だけを追加し、画像、Base64、複製した個体データを保存しない。
- 本番バンドルの増加は、画像素材を除き30KB gzip以内を目安にする。

## 19. テスト仕様

### 19.1 単体テスト

- Version 5からVersion 6へ全個体を保持して移行できる。
- 移行直後の`favoriteSpecimenIds`は空である。
- 存在する個体だけをお気に入りにできる。
- `favorite: true`の二重送信で重複しない。
- `favorite: false`の二重送信で他のIDを変更しない。
- お気に入り変更で時刻、進捗、日記、トラップ結果を変更しない。
- 新しい順が日、時刻、IDで安定する。
- 大きい順がサイズ、日、時刻、IDで安定する。
- 自己ベストとの差が0.1mm単位で正しい。
- 通常木、固定バナナ、固定ライト、プレイヤーバナナの表示名が正しい。
- 存在しないお気に入りIDだけを修復できる。
- 重複お気に入りIDを順序を保って1件へ正規化できる。
- 既存個体の木や調査ポイントが消えても個体を保持できる。
- 未発見種から個体一覧を取得できない。

### 19.2 結合テスト

- 通常木の捕獲結果からお気に入り登録し、図鑑へ表示される。
- プレイヤートラップの捕獲結果から同じ操作ができる。
- 種類一覧から個体一覧、詳細、元のカードへ戻れる。
- とっておきタブから詳細を開き、解除後に一覧から外れる。
- 種類別一覧で解除しても個体一覧からは消えない。
- 31個体で「さらに30匹見る」が1回正しく動く。
- 並び替え後に表示件数が30へ戻る。
- 捕獲直後の設定を保存・再読込して維持できる。
- 18時保留中の捕獲結果で設定後、迎えが1回だけ実行される。
- 20時直前の捕獲結果で設定後、日記と翌日に維持される。

## 20. 完了条件

次をすべて満たした時点で「とっておきの虫かご・個体コレクション Ver.5」を完成とする。

1. 発見済み種類から、その種類の全個体を見返せる。
2. 個体ごとのサイズ、日時、場所、木、調査点、捕獲元が正しい。
3. 通常木、固定トラップ、プレイヤートラップの個体を同じ仕組みで表示できる。
4. 自己ベストと差を0.1mm単位で正しく表示できる。
5. 新しい順と大きい順を安定して切り替えられる。
6. 31件以上を30件単位で追加表示できる。
7. 捕獲結果と個体詳細の両方から、とっておきを設定・解除できる。
8. 二重送信でもお気に入り状態が意図せず反転しない。
9. お気に入り変更で時間、観察進捗、抽選、サイズ、ランキング状態が変化しない。
10. とっておきタブで種類をまたいだお気に入り個体を見られる。
11. 未発見種の個体情報や正確な場所を表示しない。
12. Version 5から個体を失わずVersion 6へ移行できる。
13. 不正なお気に入り参照だけを安全に修復できる。
14. 500個体でも明確な操作停止がない。
15. 360×640で一覧、詳細、戻る、お気に入り操作が破綻しない。
16. キーボード、スクリーンリーダー、ブラウザ戻るに対応する。
17. 自動テスト、型チェック、本番ビルド、GitHub Verifyが成功する。
18. Vercel本番で「捕獲→とっておき登録→種類別一覧→個体詳細→とっておき一覧→解除→再読み込み」を実操作確認できる。
