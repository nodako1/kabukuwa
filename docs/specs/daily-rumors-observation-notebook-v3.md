# カブクワ 次期追加機能仕様書

## 日替わりの噂と夏休み観察ノート Ver.3

- 作成日: 2026-07-21
- 対象: `nodako1/kabukuwa`
- 対象環境: スマートフォン向けWebアプリ（Vercel本番環境）
- 前提: 「木を覗く採集体験・周回型ワールド Ver.2」が実装済み
- ステータス: 次期実装へ着手可能

## 1. 結論

次の追加機能では、毎朝ひとつだけ変わる「今日の自然のようす」、NPCから聞ける曖昧な噂、任意の観察テーマ、日々の行動を残す夏休み観察ノートを追加する。

Ver.2で完成した「歩いて木を見つけ、1本ずつ覗く」体験を主役のまま維持しつつ、毎日同じ順路を回るだけにならない理由を作る。噂は正解の木を示す攻略マーカーではなく、今日どこを、どのように見ようかを考えるきっかけにする。

観察テーマは達成必須のクエストにしない。未達成による損失、連続ログイン、広告視聴、レア虫の捕獲を条件にせず、「今日は根元も見てみよう」と遊び方を少し変える任意の目標にする。達成時は観察スタンプと短い日記文だけを得る。

## 2. 追加する理由

Ver.2完成後の基本ループは次のようになる。

```text
道を歩く
  ↓
気になる木へ寄る
  ↓
樹液・割れ目・根元を覗く
  ↓
虫影を見つけて捕まえる
  ↓
周回して家へ戻る
```

このループ自体は本作の核として十分に面白い。一方、日が変わっても判断材料が同じだと、効率のよい木だけを同じ順番で調べる遊びへ収束しやすい。

そこで、日ごとに次の3つを少しだけ変える。

1. 自然のようす
2. 人から聞く噂
3. 観察してみるテーマ

結果を保証せず、行き先と見方だけを揺らすことで、宝探しの不確実さを残しながら「次の日も出かける理由」を作る。

## 3. 体験目標

- 朝に「今日はどこを見よう」と小さな計画を立てられる。
- NPCとの会話が世界観だけでなく、探索のきっかけにもなる。
- 噂どおりの結果にならない日も含め、自然を相手にしている感覚がある。
- 虫が捕れない日でも、歩いた場所や見たものが記録として残る。
- 観察ノートを見返すと、自分だけの夏休みの過ごし方が分かる。
- 目標を無視して自由に虫取りしても、不利にならない。

## 4. 1日の基本フロー

```text
6:00 朝
  ↓
「今日の自然のようす」と任意の観察テーマを表示
  ↓
家を出る／おばあちゃんや道中のNPCに話しかける
  ↓
今日の噂を聞く
  ↓
自由に周回・寄り道・木の接写調査
  ↓
行動に応じて観察ノートを自動記録
  ↓
20:00 1日のまとめ
  ↓
観察テーマ達成時はスタンプと短い日記文を追加
```

朝の案内は閉じられる1枚のカードとし、閉じた後は図鑑内の「観察ノート」から再確認できる。

## 5. 今日の自然のようす

### 5.1 基本ルール

- 1日につき1種類だけ選ぶ。
- 世界シードと日数から決定的に生成する。
- 同じセーブ、同じ日では再読み込みしても変わらない。
- 直前2日と同じ種類を避ける。
- 虫のサイズへ影響させない。
- レア虫の種類や具体的な木を確定表示しない。
- 出現率を変える場合も、基礎値に対して最大1.2倍までとする。

### 5.2 初期種類

| ID | 表示名 | 朝の説明 | 主な効果 |
| --- | --- | --- | --- |
| `lively-sap` | 樹液がにぎやかな日 | 甘い匂いがいつもより強い | 樹液ポイントの環境昆虫を増やし、出現判定を最大1.15倍 |
| `quiet-roots` | 根元が気になる日 | 日が高くなると土の近くが涼しそう | 昼の根元への割当重みを上げる |
| `forest-evening` | 夕方の林に気配 | 木漏れ日が弱くなる頃、羽音が増えそう | 雑木林・クヌギ林の夕方の出現判定を最大1.15倍 |
| `sweet-breeze` | 甘い匂いが残る日 | 裏庭の仕掛けをあとで見てみよう | 夜のバナナトラップの環境昆虫と出現判定を最大1.2倍 |
| `moths-at-light` | 灯りに集まる日 | 暗くなると小さな羽音が増えそう | 夜のライトトラップの環境昆虫を増やす |
| `still-summer` | 静かな夏の日 | 目立つ気配がなくても、よく見れば何かいるかも | 出現率補正なし。「！」なしの木を覗く観察テーマを出しやすくする |

### 5.3 表現上の注意

- 効果量や倍率はゲーム画面へ数値表示しない。
- 「必ずいる」「レア虫が出る」という文言を使わない。
- 気配マーカーの判定を直接成功させない。
- 自然のようすによって環境昆虫だけが増え、採集対象は見つからない場合も残す。

## 6. 今日の観察テーマ

### 6.1 方針

1日につき1件だけ提示する、任意の行動目標とする。テーマを達成しなくても、翌日への進行、図鑑、出現率、秘密ルート、ランキングに不利益を与えない。

### 6.2 初期テーマ

| ID | 表示文 | 達成条件 |
| --- | --- | --- |
| `inspect-three-trees` | 木を3本、じっくり覗いてみよう | 異なる木の調査セッションを3件確定 |
| `look-high-and-low` | 幹と足元の両方を見てみよう | 同じ木で主要ポイントと根元を確認 |
| `trust-your-eyes` | 気配のない木も1本見てみよう | 未発見の「！」がない木を1本調査 |
| `visit-two-woods` | 2つの林を歩いてみよう | 雑木林とクヌギ林の両方へ到着 |
| `listen-to-someone` | 村の誰かの話を聞いてみよう | NPCと1回会話 |
| `check-a-trap` | 夜の仕掛けを確認してみよう | 18時以降にバナナかライトを調査 |
| `complete-one-tree` | 1本の木をすみずみまで見よう | その木の有効な全ポイントを確認 |
| `walk-the-loop` | 寄り道しながら一周してみよう | 同じ日に主周回路8フィールドへ到着 |

### 6.3 生成時の実現可能性

- 未解放の秘密ルートを必要条件にしない。
- 広告視聴を必要条件にしない。
- 特定種、レア虫、一定以上のサイズ、ランダムな捕獲成功を必要条件にしない。
- 夜まで遊ぶ必要があるテーマは、当日の自然のようすと導線が合う場合だけ候補にする。
- 旧セーブ移行直後など判定材料が不足する場合は、`inspect-three-trees`へフォールバックする。
- 直前2日と同じテーマを避ける。

### 6.4 達成報酬

- 夏休み観察ノートへ1個の観察スタンプを付ける。
- その日の行動に合った1〜2文の自動日記を追加する。
- 能力、出現率、アイテム、通貨は付与しない。
- 連続達成数と未達成日数を表示しない。

## 7. NPCの噂

### 7.1 基本ルール

- 毎日、対象NPCのうち1人が「今日の噂」を持つ。
- そのNPCへ当日最初に話しかけた際、通常会話の後へ噂を追加する。
- 噂を聞いても通常の会話進行回数は従来どおり1回だけ増える。
- 噂は当日の自然のようすと矛盾しないが、具体的な木や出現結果を保証しない。
- 対象NPCに会いにくい日は、おばあちゃんが代わりに噂を持つ。
- 噂を聞かなくても自然のようすと観察テーマは有効とする。

### 7.2 NPC別の役割

| NPC | 噂の傾向 | 例 |
| --- | --- | --- |
| おばあちゃん | 天気、時間帯、裏庭 | 「今日は甘い匂いが長く残りそうだねぇ。」 |
| 神社のおじさん | 神社、竹林、夕方 | 「夕方になると、石段の向こうで羽音がするかもしれないよ。」 |
| 虫博士 | 樹液、根元、観察方法 | 「昼は幹だけじゃなく、根元の落ち葉も見てごらん。」 |
| ライバル少年 | 場所の噂、競争心 | 「さっき林のほうで何か黒い影を見たぞ！」 |
| 駄菓子屋のおばちゃん | 村の雰囲気、子どもの目撃談 | 「学校帰りの子が、灯りに虫が集まるって言ってたよ。」 |

### 7.3 噂の精度

- フィールド単位または調査ポイント種別までを上限とする。
- 木ID、虫ID、サイズ、正確な出現時刻を明かさない。
- 噂の場所で何も捕れない結果を許容する。
- 完全な嘘にはせず、環境昆虫、自然補正、観察テーマのいずれかと関連させる。

## 8. 夏休み観察ノート

### 8.1 記録する内容

各日について次を自動記録する。

- 今日の自然のようす
- 今日の観察テーマと達成状態
- 聞いた噂と話したNPC
- 初めて訪れたフィールド
- 調査した木の数
- 見た調査ポイント種別
- 捕獲数と初捕獲種
- その日最大の個体
- 観察スタンプ
- 自動生成した短い日記文

環境昆虫は別図鑑にせず、「樹液でカナブンを見た」など、その日の観察文へだけ反映する。

### 8.2 日記文の例

```text
7月21日
今日は樹液の匂いが強い日だった。
クヌギ林で3本の木を覗き、最後に根元でクワガタの影を見つけた。
```

捕獲がなかった日も次のように記録する。

```text
今日は虫影を捕まえられなかった。
でも神社の古木で、カナブンとアリが同じ樹液に集まるところを見た。
```

### 8.3 画面導線

- 既存の「カブクワ図鑑」を2タブ構成にする。
- タブ1: 虫図鑑
- タブ2: 夏休み観察ノート
- 最新の日を先頭にする。
- 日付カードを開くと、その日の詳しい記録を表示する。
- 朝カードを閉じた後も、当日の自然のようすとテーマをここで再確認できる。
- 未達成テーマには失敗印を付けず、「今日は自由に過ごした」と表示する。

## 9. 1日のまとめ画面

既存の20:00まとめへ次を追加する。

1. 今日の自然のようす
2. 調べた木の本数
3. 見た調査ポイント
4. 今日の観察テーマの結果
5. 観察スタンプ
6. 自動日記の1文目

テーマ達成演出は短くし、捕獲数や最大個体の表示より目立たせすぎない。

## 10. 出現率・広告・ランキング

### 10.1 判定順序

出現判定は次の順序で行う。

```text
既存の基礎出現率
  ↓
今日の自然補正（自然由来、最大1.2倍）
  ↓
広告の出現率UP
  ↓
存在判定
  ↓
種類抽選
  ↓
サイズ抽選（自然補正・広告情報を渡さない）
```

### 10.2 ランキング状態

- 基礎出現または今日の自然補正によって出現した個体は、自然出現として扱う。
- 広告を適用したことで初めて追加出現した個体だけ、ランキング対象外とする。
- 同じ存在ロールを基礎、自然補正後、広告後で共有し、補正によって元からいた虫を消さない。
- サイズ抽選キーには自然補正ID、広告状態、観察テーマ達成状態を含めない。
- 観察テーマの達成で出現率を上げない。

## 12. 決定的生成

### 12.1 日次プラン

日次プランは次をキーとして生成する。

```text
rngVersion + worldSeed + day + "daily-plan"
```

生成手順:

1. 自然のようす候補を並べる。
2. 直前2日の種類を候補から除く。
3. 決定的ロールで1種類選ぶ。
4. 当日に実現可能な観察テーマだけを抽出する。
5. 直前2日のテーマを除外して1件選ぶ。
6. 当日会えるNPCから噂役を選ぶ。
7. 候補が空なら、おばあちゃんと基本テーマへフォールバックする。

生成後の`DailyPlan`をセーブし、コンテンツ更新で候補順が変わっても進行中の日の内容を変えない。

### 12.2 進捗イベント

観察進捗は既存コマンドの成功後にだけ更新する。

- 接写セッション確定
- 調査ポイント確認
- フィールド到着
- NPC会話完了
- 捕獲完了
- 接写内で環境昆虫を表示

画面を開くだけ、ボタン連打、無効なコマンドでは重複加算しない。ID集合で冪等に記録する。

## 14. UI・アクセシビリティ

- 朝カードはモーダルとしてフォーカスを閉じ込め、Escapeで閉じられる。
- 自然のようすを色だけで区別せず、名前、短文、アイコン形状を併用する。
- 観察テーマの進捗は`1 / 3`など文章でも読み上げられるようにする。
- 達成演出は`prefers-reduced-motion`で跳ねや紙吹雪を止める。
- 観察ノートは縦スクロール可能にし、タップ領域を44×44 CSS px以上にする。
- 噂文、日記文、テーマ文は小学生でも読みやすい短文を基本にする。
- 未達成を赤や警告音で表現しない。

## 17. テスト仕様

### 17.1 単体テスト

- 同じシードと日で同じ日次プランになる。
- 直前2日と同じ自然・テーマを避ける。
- 実現不能なテーマを選ばない。
- NPC候補がない場合、おばあちゃんへフォールバックする。
- 自然補正が最大1.2倍を超えない。
- 自然補正と広告補正が元からいた虫を消さない。
- 自然補正と広告がサイズを変えない。
- 広告だけで追加された個体はランキング対象外になる。
- 同じ木、ポイント、フィールドを重複加算しない。
- テーマを無視しても翌日へ進める。
- Version 3からVersion 4へ既存データを保持して移行できる。

### 17.2 結合テスト

- 朝カードを閉じた後、観察ノートで同じ内容を確認できる。
- 当日の噂役へ話しかけると通常会話と噂が1回だけ表示される。
- 木の接写、ポイント切替、周回移動がテーマ進捗へ正しく反映される。
- テーマ達成後も重複スタンプが付かない。
- 20:00のまとめで日記が確定し、翌日も見返せる。
- 再読み込みしても日次プラン、進捗、噂、日記が変わらない。

- 主人公移動中に進捗表示が操作を邪魔しない。
- `prefers-reduced-motion`で過度な達成演出がない。
