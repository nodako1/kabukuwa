# カブクワ 次期実装仕様書

## 木を覗く採集体験・周回型ワールド Ver.2

- 作成日: 2026-07-21
- 対象: `nodako1/kabukuwa`
- 対象環境: スマートフォン向けWebアプリ（Vercel本番環境）
- 前提: 「歩いて探すフィールド探索 Ver.1」が実装済み
- ステータス: 実装済み

## 1. 結論

次の実装では、現在の「木を調べると即座に結果シートが出る方式」を、木の表面を実際に覗いて虫影を探す採集体験へ置き換える。

主人公が木を調べると、樹液、幹の割れ目、根元などの接写画面へ切り替わる。接写画面にはカブトムシ・クワガタムシだけでなく、カナブン、ガ、チョウ、アリなどの採集対象外の虫も配置し、樹液に虫が集まっている空気を作る。採集対象をタップすると捕獲し、その後に種類、サイズ、自己ベストなどの詳細を表示する。

歩行フィールドでは、気配のある木へ控えめな「！」を表示する。ただし「！」は当たり確定表示にはしない。気配がなくてもレアな虫がいる場合があり、カナブンやガだけで「！」が出る場合もある。これにより、木を1本ずつ覗く宝探しを維持する。

ワールドは、おばあちゃんの家を起点とした四角い周回路へ再構成する。通常の場所移動は行動ボタンを使わず、木、柵、藪などで閉じられた外周のうち、道として空いている画面端を越えると隣のフィールドへ切り替わる。

## 2. 調査結果と採用理由

### 2.1 樹液の接写に採集対象外の虫を置く

クヌギやコナラの樹液には、カブトムシ・クワガタムシだけでなく、カナブン、チョウ、ガ、ハチなどが集まる。千葉県立中央博物館の観察記録でも、同じ樹液にカブトムシ、カナブン、クロカナブンが集まる様子が記録されている。

したがって、採集対象外の虫は単なる飾りではなく、「樹液を覗く」場面の説得力と生命感を作る要素として採用する。

### 2.2 根元を第二の調査ポイントにする

国土交通省の生物資料では、カブトムシは夜間や早朝に樹液へ集まり、昼間は木の根元の草むらや土中などにいることが示されている。

この性質は、最初に幹を見たあと「足元も見る」という追加行動を自然に成立させる。木を増やさずに発見場所を増やす本作の方針と相性がよいため、根元を正式な調査ポイントとして採用する。

### 2.3 「！」を当たり確定にしない

探索の面白さには、「何かありそうだが、結果はまだ分からない」という情報差が必要になる。ゲーム研究でも、不確実さと将来の報酬への期待のバランスが、好奇心に基づく探索を支えるとされている。

そのため「！」は正解表示ではなく、羽音、葉の揺れ、樹液のにおいなどの曖昧な手掛かりとして扱う。これは上記研究から本作向けに導いた設計判断である。

### 2.4 場所の切替は画面端の道で行う

通常移動では、通れる道を見つけて自分の足で境界を越える方が、歩行と場所移動を一続きに感じられる。閉じた画面端には必ず見える障害物を置き、通れる箇所だけを道として開ける。

場面切替はReactの同一ページ内で行う。短い暗転はCSSで必ず動作させ、対応ブラウザではView Transition APIを段階的な強化として利用できる構造にする。

## 3. 体験上の目標

### 3.1 プレイヤーに感じてほしいこと

- 木へ近づくまで、何がいるか分からない。
- 気配を見つけたとき、「もしかしたら」と期待できる。
- 幹にはいなくても、割れ目や根元に隠れているかもしれない。
- 接写した瞬間に、樹液へ集まる虫の世界を覗き込んだように感じる。
- 虫影を自分で見つけてタップしたことで、「見つけた・捕まえた」と感じる。
- 寄り道しながら歩いていると、自然に家の近くへ戻ってくる。

### 3.2 維持する既存方針

- 採集対象は世界中のカブトムシ・クワガタムシだけとする。
- カナブン、ガ、チョウなどは図鑑や捕獲数へ入れない。
- 捕獲操作に失敗判定を入れず、虫影を正しくタップすれば必ず捕まる。
- 広告効果は出現率、効果時間、追加ヒントだけに限定する。
- サイズは広告の影響を受けず、完全に運で決める。
- 18:00の迎え、18:00〜20:00の家・裏庭制限、秘密ルート16:00〜18:00を維持する。
- 歩行だけでは時間を進めず、調査、会話、場所移動で時間を進める。

## 6. 周回型ワールド

### 6.1 接続構成

既存の場所と道中フィールドを、次の主周回路へ組み直す。

```text
       雑木林 ─── クヌギ林 ─── 小学校
          │                         │
         林道                      竹林
          │                         │
おばあちゃんの家 ─ 田んぼ道 ─── 神社
          │                         │
        裏庭                 秘密の小道 ─ 秘密の森
```

主周回路は次の8接続で構成する。

1. おばあちゃんの家 ↔ 田んぼ道
2. 田んぼ道 ↔ 神社
3. 神社 ↔ 竹林
4. 竹林 ↔ 小学校
5. 小学校 ↔ クヌギ林
6. クヌギ林 ↔ 雑木林
7. 雑木林 ↔ 林道
8. 林道 ↔ おばあちゃんの家

裏庭は家からの短い枝道、秘密の小道と秘密の森は神社からの条件付き枝道とする。

### 6.2 設計ルール

- 主周回路は時計回り、反時計回りのどちらでも一周できる。
- 主周回路には行き止まりを作らない。
- 寄り道用の裏庭と秘密の森だけは、入った道を戻る構造でよい。
- 隣接フィールドへの移動時間は原則5分とする。
- 主周回路を一周した場合の移動時間は合計40分を基準とする。
- 各出口付近には看板、道の向き、遠景など、行き先が分かる手掛かりを置く。
- 地図には主周回路を線で表示し、現在地と発見済みの場所だけを強調する。
- 現在の林道の「全場所へ分岐するハブ」構造は廃止し、周回路の1区間へ変更する。

## 7. 画面端によるフィールド切替

### 7.1 通れる場所の表現

- フィールド外周は、木、藪、柵、石垣、建物、水路などの当たり判定で閉じる。
- 次のフィールドへ続く場所だけ、主人公1.5〜2人分以上の幅で障害物を切る。
- 開口部は土の道、石段、門、橋などで視覚的に示す。
- 通れそうに見える場所へ透明な壁だけを置かない。
- 通れない条件がある出口は、条件に合う障害物を実際に表示して塞ぐ。

例:

- 18:00以降の遠方出口: 門、柵、おばあちゃんの呼び止め
- 未解放の秘密の小道: 深い藪
- 秘密ルートの時間外: 夕闇の藪、立入をためらう演出

### 7.2 自動切替条件

次をすべて満たしたときだけフィールドを切り替える。

1. 主人公が有効な開口部の範囲内にいる。
2. 主人公が外向きに移動している。
3. 主人公の当たり判定中心が境界トリガー線を越えた。
4. 出口の時間・解放条件を満たしている。
5. 会話、接写、結果、18時イベントなどで入力がロックされていない。

通常出口は行動ボタンの対象から外し、行動ボタンは木、NPC、報酬地点などに専念させる。

### 7.3 切替演出

1. 入力を停止する。
2. 180〜250msで画面を暗くする。
3. 移動時間を進め、接続先フィールドを読み込む。
4. 接続先の反対側の端から56〜72px内側へ主人公を配置する。
5. 場所名を短く表示する。
6. 180〜250msで画面を戻す。
7. 入力を再開する。

`prefers-reduced-motion` が有効な場合は、移動を伴うアニメーションを使わず短い透明度変化だけにする。

### 7.4 誤遷移・往復防止

- 到着地点は境界トリガーの内側へ置き、接触したままにしない。
- 切替後350〜500msは次の境界判定を無効にする。
- 時間だけに頼らず、「到着側トリガーから一度離れた」ことを再発火条件にする。
- 同じフレームで複数の出口が発火しないよう、切替処理に一意なトークンを持たせる。
- 接続先の安全座標が障害物内なら、定義済みの代替スポーン地点を使う。

### 7.5 閉じた出口へ触れた場合

- 主人公は障害物で停止し、フィールド切替は行わない。
- 「もう暗いから遠くへは行けない」など、理由を1回だけ短く表示する。
- 同じ場所を押し続けても通知を連続表示せず、3秒の再表示待ちを設ける。

## 8. 木の調査モデル

### 8.1 木と調査ポイント

現在の1ホットスポットを、1本の木と、その木に属する1〜3個の調査ポイントへ置き換える。

調査ポイント種別:

- `sap`: 樹液
- `bark-crack`: 幹の割れ目・うろ
- `root`: 根元・草むら・落ち葉
- `banana-trap`: バナナトラップ
- `light-trap`: ライトトラップ

### 8.2 木の基本パターン

すべての木へ同じ数のポイントを付けず、見た目と攻略感に差をつける。

| 木の型 | 最初に見る場所 | 追加で見られる場所 | 用途 |
| --- | --- | --- | --- |
| 樹液の木 | 樹液 | 根元 | 昼夜の基本採集 |
| 古木 | 幹の割れ目 | 根元 | 隠れたクワガタ向け |
| トラップの木 | バナナ／ライト | 幹または根元 | 裏庭の夜採集 |
| 小さな木 | 幹 | なし | 短い空振りも作る |

### 8.3 木の本数

- Ver.2では原則として現在の木の本数を増やさない。
- 既存の木ホットスポットを、調査ポイント付きの木へ変換する。
- 重要な木に2〜3ポイント、その他の木に1ポイントを持たせる。
- 1画面内で木のマーカーが重ならない密度を維持する。
- 調査ポイントの位置と順序は固定し、虫のいるポイントだけを変える。

## 9. 調査セッションと時間

### 9.1 15分を進めるタイミング

- フィールドで「木を調べる・15分」を実行した時点で、ゲーム内時間を15分進める。
- 同じ木の接写内で、樹液、割れ目、根元を切り替えても追加時間を進めない。
- 採集対象をタップしても追加時間を進めない。
- 接写を閉じて同じ調査セッションへ戻っても、再度15分を請求しない。

これにより、調査ポイントが多い木だけ時間的に不利になることを防ぐ。

### 9.2 抽選単位

- 木へ近づいたときの気配表示と、木を開いた後の接写が食い違わないよう、木の全調査ポイントを返す純粋な「調査プラン生成関数」を使う。
- 気配判定時は同じ関数の結果を参照するだけとし、時間消費や調査済み状態の保存は行わない。
- 木を開いた時点で同じ調査プランをセッションとして確定し、15分の時間消費と一緒に保存する。
- 同じ日、訪問番号、時間帯、木IDでは同じ結果になる決定的抽選を使う。
- セッションの時間帯は調査開始時を使い、15分後に時間帯境界を越えても開いている接写の内容は変えない。
- 調査ポイントを切り替えるたびに再抽選しない。
- 再読み込み、接写の閉じ直し、戻る操作で結果を変えない。
- Ver.2では1本の木の1調査セッションにつき、捕獲可能な虫は最大1匹とする。
- 捕獲可能な虫がいないポイントにも、環境昆虫は表示できる。

最大1匹にすることで、調査ポイントを増やしても現在の出現率と捕獲数のバランスを大きく変えない。複数捕獲は出現率調整後の将来拡張とする。

### 9.3 時間帯が変わった場合

- 朝、昼、夕方、夜の区切りを越えたら、新しい時間帯の調査セッションとして生成できる。
- 接写を閉じて同じ木を開き直しても、現在の訪問中は同じセッションを保持する。
- 別フィールドから戻って訪問番号が増えた場合は、新しい調査セッションを生成する。
- 18:00へ到達する調査は接写と捕獲結果を最後まで表示し、接写を閉じたあとに迎えイベントを開始する。
- 20:00へ到達する裏庭の調査も接写を最後まで表示し、その後に一日のまとめへ進む。

## 10. 接写画面

### 10.1 画面構成

```text
┌─────────────────────┐
│  クヌギの木            18:05 │
├─────────────────────┤
│                           │
│   カナブン      樹液       │
│          虫影             │
│      ガ                   │
│                           │
├─────────────────────┤
│ [足元も見る] [木から離れる] │
└─────────────────────┘
```

- フィールドの歩行UIと仮想スティックは隠す。
- 上部に木の名前、現在の調査ポイント、調査後の時刻を表示する。
- 中央を接写画像と虫の表示領域にする。
- 下部に追加ポイントと「木から離れる」を表示する。
- 戻ったときは、調査開始前と同じ木の前、同じ向きへ主人公を戻す。

### 10.2 表示順

1. 180〜250msの暗転
2. 接写背景を表示
3. 葉の揺れ、樹液の光、環境昆虫を表示
4. 捕獲可能な虫影を表示
5. 約0.8秒後に補助ボタンを表示

最初から「何もいない」と文字で断定せず、一度自分の目で画面を探す時間を作る。

### 10.3 空振り

- 捕獲可能な虫がいない場合も、環境昆虫や樹液の動きを表示する。
- 0.8〜1.2秒後に「カブト・クワガタはいないようだ」と控えめに表示する。
- 追加ポイントがある場合は、空振り表示の後も「足元も見る」を選べる。
- すべての調査ポイントを見終えた場合だけ、「この木は調べ終えた」と表示する。

## 11. 捕獲操作と結果

### 11.1 捕獲対象

- 捕獲できる虫だけに、最低48×48 CSS pxのタップ領域を持たせる。
- 虫の画像より当たり判定を少し広げ、小さい画面でも押しやすくする。
- 最初の1.5〜2秒は強い誘導を出さず、見つける時間を作る。
- その後も見つからない場合だけ、羽の動きや小さな光で控えめに誘導する。
- アクセシビリティ名は捕獲前に種類を明かさず、「虫影を捕まえる」とする。

### 11.2 環境昆虫

- 環境昆虫は捕獲ボタンにせず、コレクションへ追加しない。
- 捕獲対象と誤認しにくい大きさ、明度、動きにする。
- タップを奪わないよう、原則として`pointer-events: none`にする。
- カブトムシ・クワガタムシを環境昆虫として偽表示しない。

### 11.3 捕獲演出

1. 虫影をタップする。
2. 他の入力をロックする。
3. 網、光、羽音など350〜600msの短い演出を出す。
4. 捕獲済みIDを先に保存対象へ反映する。
5. 詳細結果シートを表示する。

連打しても同じ個体を2回追加しない。

### 11.4 詳細結果

結果シートには次を表示する。

- 虫の画像
- 和名
- カブトムシ／クワガタムシの分類
- サイズ
- 初捕獲
- 自己ベスト更新
- 捕獲場所、木、調査ポイント
- 捕獲日と時刻
- ランキング対象／広告補助による対象外
- 図鑑へ追加されたこと

「閉じる」で元の接写へ戻り、未確認の調査ポイントがあれば続けて見られる。

## 12. 環境昆虫の配置

### 12.1 初期種類

| 場面 | 主な環境昆虫 | 表現 |
| --- | --- | --- |
| 昼の樹液 | カナブン、クロカナブン、チョウ、アリ | ゆっくり吸蜜、歩行 |
| 夕方の樹液 | カナブン、ガ、羽虫 | 羽ばたき、出入り |
| 夜の樹液 | ガ、羽虫、カナブン | 暗所の小さな動き |
| 根元 | アリ、ダンゴムシ相当の小さな影 | 落ち葉の下を動く |
| ライトトラップ | ガ、羽虫 | 光の周囲を回る |
| バナナトラップ | カナブン、アリ | 餌の周囲へ集まる |

スズメバチは樹液へ来る現実性がある一方、恐怖表現と安全メッセージが必要になるため、Ver.2初期対象外とする。

### 12.2 時間とポイントの整合

- 昼の根元は、夜よりもカブトムシの隠れ場所になりやすく設定できる。
- 夜の樹液とライトトラップは、環境昆虫の数と動きを増やす。
- 朝は樹液と根元の両方に可能性を残す。
- 外国産カブトムシは既存ルールどおり夜の裏庭を中心とする。
- 現実を完全再現せず、昼のレア出現率も0にはしない。

## 13. フィールド上のマーカー

### 13.1 マーカー種類

| 表示 | 意味 | 表示条件 |
| --- | --- | --- |
| `！` | 何かの気配 | 木の近くで気配判定に成功 |
| トラップ記号 | トラップがある | 設置済み・固定トラップの木 |
| 小さなチェック | 今回は調べ終えた | 全調査ポイントを確認済み |
| なし | 未確認または気配なし | 標準状態 |

### 13.2 気配は正解表示にしない

気配判定は、木の調査セッションと同じシードから決定的に生成する。初期調整値は次を基準とし、データ定義から変更可能にする。

| 木の状態 | 「！」の基準表示率 |
| --- | ---: |
| 捕獲可能な虫がいる | 55% |
| 捕獲対象はいないが環境昆虫が活発 | 20% |
| 目立つ活動がない | 5% |

したがって、次の両方が必ず成立する。

- 「！」がない木にも捕獲可能な虫がいる場合がある。
- 「！」がある木にカナブンやガしかいない場合がある。

レア度が高いほど「！」を出しやすくする処理は入れない。レアな虫を確実に特定できないことを優先する。

### 13.3 表示距離

- 「！」は木から220px以内へ近づいたときに初めて表示する。
- 一度見つけた「！」は、その調査セッション中は木から離れても保持する。
- 画面外や地図全体へ、未発見の「！」を一覧表示しない。
- 初回表示時だけ小さく跳ね、その後は静かな点滅にする。
- `prefers-reduced-motion` では点滅と跳ねを止める。

### 13.4 トラップ記号

- 裏庭のバナナトラップ、ライトトラップは固定位置へ表示する。
- 利用可能なトラップは暖色、時間外は灰色で表示する。
- トラップ記号は当たりを示さず、「ここに仕掛けがある」ことだけを示す。
- 将来の自由設置へ備え、マーカー状態は木データではなくセーブ側からも供給できる設計にする。

## 14. 出現率・広告・ランキング

### 14.1 出現抽選

- 既存の虫ごとの場所、時間帯、スポット種別、出現率を引き継ぐ。
- 木単位で出現抽選を1回行い、当選した虫を適切な調査ポイントへ割り当てる。
- 調査ポイント数を増やしても、出現抽選回数を増やさない。
- 樹液、割れ目、根元の適性は、当選後の割当重みとして使う。

### 14.2 広告効果

- 出現率UPは木単位の出現抽選だけに作用する。
- 気配表示率を広告ボタンから直接上げない。
- 出現率UPで虫が配置された結果として、間接的に「！」が出やすくなることは許可する。
- 効果時間UPとおばあちゃんの追加ヒントは既存仕様を維持する。
- サイズ抽選へ広告情報を渡さない。
- 広告補助で出現した個体のランキング対象外表示を維持する。

## 15. 18時・夜・秘密ルート

### 15.1 18:00の迎え

- 遠方で移動または調査により18:00へ到達した場合、進行中の短い接写・結果表示を完了させる。
- 接写を閉じた直後に「おーい！迎えに来たよ！」を表示する。
- 暗転後、おばあちゃんの家の玄関前へ移動し、18:15にする。
- 迎え後は、家から田んぼ道・林道へ続く開口部を障害物で閉じる。
- 家と裏庭の往復は画面端移動で継続できる。

### 15.2 18:00〜20:00

- 裏庭の樹液、バナナトラップ、ライトトラップを接写調査できる。
- 夜用の背景色、環境昆虫、光の表現へ切り替える。
- 20:00へ到達する調査は結果を確認した後、一日のまとめへ進む。

### 15.3 秘密ルート

- 神社のおじさんとの会話条件と16:00〜18:00の両方を満たすと、藪が開いて道が見える。
- 条件未達または時間外は、同じ場所へ当たり判定付きの藪を表示する。
- 地図には解放前の秘密ルートを表示しない。
- 17:55など移動後に18:00を越える場合は、接続先へ一瞬到着させず迎え処理を優先する。

## 16. データ設計

### 16.1 木と調査ポイント

```ts
type InspectionSceneKind =
  | "sap"
  | "bark-crack"
  | "root"

  | "light-trap";

interface TreeInspectionPoint {

  activePeriods?: TimePeriod[];

}

interface TreeDefinition {
```

### 16.2 画面端出口

```ts
interface EdgeExit {
  id: string;
  side: "top" | "right" | "bottom" | "left";
  rangeStart: number;
  rangeEnd: number;
  toFieldId: FieldId;
  toSpawnId: string;
  travelMinutes: number;

}
```

`rangeStart`と`rangeEnd`は、上下端ではX座標、左右端ではY座標を表す。出口座標を1点で持つ現在の`FieldExit`から、幅を持つ`EdgeExit`へ移行する。

### 16.3 調査セッション

```ts
interface TreeInspectionSession {
  id: string;

  visitIndex: number;
```

セッションIDは世界シード、日、訪問番号、時間帯、木IDから安定して生成し、再読み込みで変化させない。

## 17. セーブSchema Version 3

### 17.1 追加保存項目

- 現在開いている接写の木IDと調査ポイントID
- 木ごとの調査セッション
- 確認済み調査ポイントID
- 捕獲済み遭遇ID
- 発見済みの気配マーカー
- トラップ状態
- 切替後の安全なフィールド位置
- 画面端切替中かどうかを復旧するための最小状態

### 17.2 Version 2からの移行

- 図鑑、捕獲個体、NPC会話回数、広告効果、秘密ルート、時刻を保持する。
- 既存の`searchedSpotIds`は、対応する木の主要調査ポイントを確認済みとして移行する。
- 旧ホットスポットIDと新しい木IDの対応表を明示的に持つ。
- 周回路変更後に現在座標が障害物内になる場合、同じフィールド内の最寄り安全地点へ移す。
- フィールド接続自体が変わっても、現在のフィールドIDは可能な限り保持する。
- 接写途中の保存がないVersion 2では、通常フィールド画面から開始する。
- 現在セーブをバックアップしてからVersion 3を保存する。
- 読み込み失敗時はバックアップを試し、ゲームを停止させない。

## 18. UI・アクセシビリティ

- 接写画面、捕獲結果、会話中は歩行入力を完全に停止する。
- 接写画面のボタンは最低44×44 CSS px、虫影のタップ領域は最低48×48 CSS pxとする。
- 色だけでマーカー状態を区別せず、形とラベルも変える。
- スクリーンリーダーへ、接写場所、捕獲可能な虫影の有無、結果を順序どおり通知する。
- 捕獲前に虫の種類を読み上げない。
- 環境昆虫は操作対象としてフォーカスさせない。
- Escapeまたはブラウザの戻る操作は、捕獲演出中以外なら確認なしでフィールドへ戻せる。
- 戻る操作でブラウザ全体を終了しないよう、接写状態をアプリ内履歴として扱う。
- 仮想スティック領域だけ`touch-action: none`とし、接写や図鑑は縦スクロールを維持する。

## 19. アート・音・演出要件

### 19.1 接写背景

初期実装では次の4系統を用意する。

1. 樹液がにじむ幹
2. 幹の割れ目・うろ
3. 木の根元と落ち葉
4. 裏庭のトラップ（バナナ／ライト差分）

- 背景と虫を別レイヤーにする。
- 虫の配置は0〜1の正規化座標でデータ化し、画面比率が変わっても幹から外れないようにする。
- 捕獲対象は透明背景の個別素材にする。
- 環境昆虫は2〜4種類、合計2〜6匹程度を基準とする。
- 画面を埋め尽くさず、捕獲対象を自分で探せる余白を残す。

### 19.2 音

Ver.2の必須条件にはしないが、音を追加する場合は次を優先する。

- 接写開始時の環境音の変化
- 小さな羽音
- 捕獲時の網の音
- 夜のライトトラップの低い環境音

音なしでも手掛かりが分かるよう、視覚表示を必ず併用する。

## 20. 性能要件

- フィールド歩行中は従来どおり60fpsを目標とし、最低45fpsを下回り続けない。
- 接写素材は木へ近づいた時点で先読みし、全種類を起動時に読み込まない。
- 接写を閉じた後、不要な大型画像とアニメーションを解放する。
- 画面外の環境昆虫アニメーションを停止する。
- CSS TransformとOpacityを中心にアニメーションし、レイアウト再計算を増やさない。
- View Transition APIを必須依存にせず、未対応時はCSS暗転へフォールバックする。
- 低性能端末向けに、環境昆虫数とパーティクル数を減らせる設定値を持つ。

## 22. テスト仕様

### 22.1 単体テスト

- 木単位で出現抽選が1回だけ行われる。
- 調査ポイント切替で出現率と時間が増えない。
- 同じシード、日、訪問、時間帯、木IDで同じセッションになる。
- 気配ありの空振りと、気配なしの当たりが生成される。
- 広告効果がサイズ抽選へ影響しない。
- 同じ遭遇IDを連打しても捕獲が1回だけ記録される。
- 環境昆虫が図鑑と捕獲個体へ追加されない。
- すべての主周回路接続に逆方向の接続がある。
- すべての安全スポーンから出口開口部へ到達できる。
- 閉じた出口の開口部に当たり判定が生成される。
- 画面端切替が1回だけ発火し、到着直後に戻らない。
- Version 2からVersion 3へ既存データを保持して移行できる。

### 22.2 結合テスト

- 家から時計回り・反時計回りの両方で一周して家へ戻れる。
- 開口部以外の画面端を通り抜けられない。
- 通れる画面端を越えるだけで隣のフィールドへ移る。
- 木を調べると15分だけ進み、接写へ切り替わる。
- 樹液を見た後に「足元も見る」で根元へ切り替わる。
- 虫影をタップすると捕獲演出後に正しい詳細が出る。
- 接写を閉じると同じ木の前へ戻る。
- 再読み込み後も同じ接写、同じ虫、同じ捕獲状態を再現する。
- 17:55の調査結果を閉じた後に迎えイベントが始まる。
- 18:00以降、家から遠方へ続く道が物理的に塞がる。
- 夜の裏庭ではライトトラップ記号と夜用接写が表示される。
- 秘密ルートは解放条件と16:00〜18:00を満たす場合だけ通れる。

### 22.3 画面確認

- 360×640
- 390×844
- 430×932
- PCキーボード操作
- 虫影とボタンが指で押しやすい。
- マーカーが木や主人公を過度に隠さない。
- 接写の背景と虫が各画面比率でずれない。
- 場所切替中に二重入力できない。
- 接写や結果表示中に主人公が動かない。
- `prefers-reduced-motion`で過度な点滅や移動がない。

## 23. 完了条件

次をすべて満たした時点で「木を覗く採集体験・周回型ワールド Ver.2」を完成とする。

1. 木を調べると接写へ切り替わり、樹液・割れ目・根元などを見られる。
2. 接写には場面と時間帯に合う採集対象外の虫が表示される。
3. カブトムシ・クワガタムシの虫影をタップして捕獲できる。
4. 捕獲後に種類、サイズ、自己ベスト、ランキング状態が表示される。
5. 1本の木に複数の調査ポイントを持たせても時間と抽選回数が増えない。
6. 「！」が当たり確定にならず、木を覗く意味が残る。
7. トラップの木をフィールド上で識別できる。
8. 主周回路を両方向から一周して家へ戻れる。
9. 通れない画面端は見える障害物で塞がれている。
10. 有効な開口部を越えると、誤往復なしで次のフィールドへ切り替わる。
11. 18時、夜の裏庭、秘密ルート、広告公平性が維持される。
12. Version 2のセーブを失わずVersion 3へ移行できる。
13. 自動テスト、型チェック、本番ビルドが成功する。
14. Vercel本番環境で主要スマートフォン幅の動作を確認できる。
