## 1. 結論

次の段階では、新しい虫、抽選、アイテム、経済機能を追加せず、現在の仮素材を統一された本番アートへ差し替え、夏の田舎を感じられる環境音と効果音を追加する。

Ver.5までで、次の遊びの循環は成立している。

~~~text
村を歩く
  ↓
気になる木を見つける
  ↓
樹液・割れ目・根元を覗く
  ↓
虫を捕まえる
  ↓
観察ノートと個体記録へ残す
  ↓
翌朝トラップを見に戻る
  ↓
好きな個体を「とっておき」にする
~~~

一方、現行画面は主人公、NPC、木、地形、接写、虫の多くをCSS図形で表現しており、音声も未実装である。遊びの仕組みをさらに増やす前に、完成した循環へ視覚・聴覚の手応えを与え、「機能を確認する試作品」から「夏休みを過ごしたくなるゲーム」へ品質を引き上げる。

本機能はゲームルールと完全に分離する。アート、音、設定状態は、虫の有無、種類、サイズ、出現率、ランキング対象状態、日次プランへ一切影響させない。

## 2. この段階で実施する理由

### 2.1 基本ループは成立している

Ver.1からVer.5までで、歩行探索、接写、複数日、日替わり要素、トラップ再訪、個体コレクションが接続された。現在不足しているのは遊び方ではなく、次の感覚である。

- 場所へ入った瞬間に神社、田んぼ、竹林だと分かる。
- 朝、昼、夕方、夜の空気が変わったと感じる。
- 木へ近づいたとき、実際に樹液を覗く緊張がある。
- 虫影を押したとき、網を振った手応えがある。
- 18時に迎えが来たとき、一日が終わる寂しさがある。
- 捕まえた個体を図鑑で見返したくなる。

### 2.2 後続機能でも素材を再利用する

今後想定する夏休みの節目イベント、思い出カード、観察ノート演出は、主人公、NPC、虫、背景、環境音を再利用する。先にイベントを増やすと、仮素材を前提に追加した画面を再調整する必要がある。本段階で描画・音声基盤を確立し、以降の実装コストを抑える。

### 2.3 現行データ構造を活用できる

現行には、フィールド座標、オブジェクト種別、木の種類、接写種別、虫ID、時間帯がデータとして分離されている。ただし既存の`sceneAssets`は`LocationId`単位で、現在の11フィールド描画には接続されていない。本仕様では、`FieldId`と各データ型を網羅する本番用Asset Manifestへ置き換える。

## 3. 体験目標

- 文字を読まなくても、現在地と時間帯を大まかに把握できる。
- 主人公の向き、歩ける道、通れない障害物を見た目から理解できる。
- 雑木林、クヌギ林、林道、秘密の森の違いを感じられる。
- 接写背景と虫を別レイヤーにし、自分で虫影を探す体験を維持する。
- 捕獲対象5種を一目で見分けられるが、捕獲前の文章では種類を確定表示しない。
- カナブン、ガ、チョウ、アリなどが樹液へ集まる生命感を表現する。
- BGMに頼らず、蝉、風、水、葉、夕方の虫などで夏の空気を作る。
- 無音でも全機能を利用できる。
- 音、動き、素材読込状況によって抽選結果が変化しない。
- 低速回線や素材読込失敗時も、現在のCSS表示で最後まで遊べる。

## 5. アート方針

### 5.1 基本スタイル

- 懐かしい日本の夏休みを感じる、温かい手描き2D。
- 斜め見下ろしを基本とし、現在のワールド座標と相性のよい遠近感にする。
- 写実的すぎず、子どもが虫を怖がりにくい柔らかな表現にする。
- 木、道、建物、虫の輪郭はスマートフォンでも判別できる太さにする。
- 現在の紙、緑、土、夕焼けの配色を基礎にする。
- 既存作品、アニメ、ゲーム、存命作家の固有画風を模倣しない。
- 画像内へ虫名、場所名、ボタン文言などの文字を焼き込まない。
- UI上の文章はすべてHTMLで表示する。

### 5.2 時間帯

| 時間帯 | 視覚方針 | 音の方針 |
| --- | --- | --- |
| 朝 | 青みのある柔らかな光、短めの影 | 鳥、弱い蝉、朝風 |
| 昼 | 明るい緑、強い日差し、道の視認性を優先 | 蝉、葉、場所固有音 |
| 夕方 | オレンジの斜光、長い影 | ヒグラシ、弱い風 |
| 夜 | 藍色、家やライト周辺の局所光 | コオロギ、夜風、ライト周辺の羽音 |

時間帯ごとに全背景を複製せず、基本素材へ色、光、影、局所照明レイヤーを重ねる。夜でも歩ける道、木、出口、行動ボタンの視認性を失わず、暗さだけで時間帯を表現しない。

### 5.3 結果を漏らさない表現

- 木の本番画像だけで虫の有無を判別できない。
- 気配「！」が出ていない木へ、虫影や特別な樹液光を追加しない。
- 気配がある場合も、捕獲対象の存在を確定する絵や音を出さない。
- プレイヤートラップのWaiting、Ready、Openedは状態マーカーで区別する。
- Readyマーカーは虫の有無、種類、サイズを示さない。
- 羽音は環境昆虫でも発生し、捕獲対象確定の合図にしない。

## 6. 基本UX

### 6.1 タイトル画面

1. 本番キービジュアルと既存タイトルを表示する。
2. 「つづきから」「夏休みをはじめる」を従来どおり表示する。
3. 近くに「音 オフ／オン」の明示的な切替を表示する。
4. 初期値はオフとし、ページ表示だけでは音を再生しない。
5. ユーザーがオンを押した時点でAudioContextの開始を試みる。
6. 開始できた場合は短く穏やかな確認音を1回だけ鳴らす。
7. ゲーム開始後、現在フィールドに合う環境音へ移る。
8. タイトルの「第一段階 プレイアブル版」を削除する。

音をオンにしなくても、追加ダイアログなしでゲームを開始できる。

### 6.2 フィールド

1. CSSフォールバックを即時表示する。
2. 現在フィールドの必須画像だけを読み込む。
3. 画像の`decode()`成功後に本番レイヤーを表示する。
4. 通信待ち中も主人公を操作可能にする。
5. 隣接フィールドの必須素材だけを低優先度で先読みする。
6. フィールド切替時は既存の暗転を維持し、環境音をクロスフェードする。
7. 木、NPC、出口、マーカー、操作ボタンは本番背景より常に判別できるようにする。

本番画像の表示開始でDOM寸法、当たり判定、カメラ位置を変化させない。

### 6.3 木の接写

1. 接写を開いた時点の`TreeInspectionSession.period`を使って光を決定する。
2. 樹皮背景と虫素材を別レイヤーで表示する。
3. 環境昆虫と捕獲対象を既存の正規化座標へ配置する。
4. 捕獲対象の透明なタップ領域を最低48×48 CSS pxで維持する。
5. フィールド環境音を弱め、接写用の葉音・羽音へ移行する。
6. 虫影を押した後だけ、網と捕獲成功の効果音を再生する。
7. 捕獲結果表示中は接写環境音をさらに弱める。
8. 結果を閉じると同じ接写音へ戻し、接写を閉じるとフィールド音へ戻す。

画像が読み込めない場合も、既存のCSS樹皮と虫影を使用する。

### 6.4 プレイヤートラップ接写の時間帯修正

現行の`PlayerTrapInspectionView`には`period-morning`と「翌朝の接写」が固定されている。Ready状態は翌朝以降なら昼・夕方にも確認できるため、本仕様で修正する。

- 固定の`period-morning`を廃止する。
- 初回確認時は`openedAtMinutes`から時間帯を決める。
- 再確認時は保存済み`openedAtMinutes`の時間帯を維持する。
- 17:55に開いて18:00の迎えが保留された場合、17:55時点の夕方表現を維持する。
- 読み上げは「バナナトラップの接写」とし、「翌朝」を固定表示しない。
- 見た目の修正でトラップ結果を再生成しない。

### 6.5 捕獲結果と個体記録

- 捕獲結果では虫種ごとの大型透明画像を表示する。
- 虫図鑑、種類別個体一覧、個体詳細、とっておき一覧では同じ虫マスター画像を再利用する。
- 一覧用には小さい表示サイズを指定し、別の巨大画像を取得しない。
- 自己ベスト、とっておき、ランキング状態は従来どおりHTMLで表示する。
- とっておき操作で捕獲音や演出を再実行しない。
- 保存済みの捕獲結果を再読込しただけでは捕獲音を再生しない。

### 6.6 ゲーム内設定

フィールド上部へ44×44 CSS px以上の設定ボタンを追加する。設定を開く前に現在位置を保存し、表示中は歩行入力を止める。

設定画面には次を表示する。

1. 音 オン／オフ
2. 環境音音量
3. 効果音音量
4. 動きを減らす
5. 操作説明をもう一度見る
6. 設定を初期値へ戻す
7. 閉じる

接写、捕獲結果、18時イベント、一日のまとめでは設定シートを重ねない。ただし、音を直ちに止められる簡易ミュートボタンは各画面から利用可能にする。

「操作説明をもう一度見る」はUIローカル状態として開き、既存の`fieldTutorialSeen`を`false`へ戻さない。操作説明の再表示でゲームセーブの`revision`を増やさない。

## 8. Visual Asset Manifest

### 8.1 ファイル配置

~~~text
src/assets/production/
  visual/
    title/
    fields/
    objects/
    characters/
    trees/
    inspections/
    insects/
    ambient-insects/
    events/
  audio/
    ambience/
    effects/
~~~

Viteのimportを通し、ビルド後は内容ハッシュ付きファイル名で配信する。コンポーネントへ実ファイルパスを直接記述しない。素材の生成元、ライセンス、用途は`docs/art/style-guide.md`と`docs/art/asset-provenance.md`へ記録する。

### 8.2 型定義

~~~ts
type VisualAssetKind =
  | "title"
  | "field-ground"
  | "field-atmosphere"
  | "field-object"
  | "character-sheet"
  | "tree"
  | "inspection-background"
  | "catchable-insect"
  | "ambient-insect"
  | "event";

type VisualLoadGroup =
  | "critical"
  | "current-field"
  | "neighbor-field"
  | "inspection"
  | "collection"
  | "lazy";

interface RasterSource {
  src: string;
  mimeType: "image/webp" | "image/png";
  width: number;
  height: number;
}

interface VisualAssetDefinition {
  id: string;
  kind: VisualAssetKind;
  sources: RasterSource[];
  fallbackClass: string;
  loadGroup: VisualLoadGroup;
  anchor?: { x: number; y: number };
  renderMode?: "contain" | "cover" | "repeat-x" | "repeat-y";
  licenseId: string;
}
~~~

`anchor`は0〜1の正規化値とし、キャラクター、木、NPCは足元または幹の根元を基準にする。

### 8.3 網羅マップ

~~~ts
const fieldArtById = {
  // 全11 FieldId
} satisfies Record<FieldId, FieldArtDefinition>;

const fieldObjectAssetByKind = {
  // 全FieldObjectKind
} satisfies Record<FieldObjectKind, VisualAssetId>;

const inspectionBackgroundByKind = {
  // 全InspectionSceneKind
} satisfies Record<InspectionSceneKind, VisualAssetId>;

const catchableInsectAssetById = {
  // 全InsectId
} satisfies Record<InsectId, VisualAssetId>;

const ambientInsectAssetById = {
  // 全AmbientInsectId
} satisfies Record<AmbientInsectId, VisualAssetId>;
~~~

ID追加時に素材不足を型検査またはManifest検査で検出する。`LocationId`だけを対象とする現行`sceneAssets`は新Manifestへ置き換え、道中3フィールドを含む`FieldId`単位へ拡張する。フォールバック確認が完了するまで既存CSSを削除しない。

## 9. フィールド描画

### 9.1 レイヤー順

~~~text
時間帯の空・地面
  ↓
非衝突の装飾
  ↓
FieldObject
  ↓
木・NPC
  ↓
主人公
  ↓
状態マーカー
  ↓
フィールド説明・操作UI
~~~

- 非操作画像へ`pointer-events: none`を設定する。
- 主人公、NPC、木の前後関係は足元Y座標で決める。
- マーカーと操作UIは背景画像より常に上へ置く。
- 画像の透明部分を当たり判定として使用しない。

### 9.2 当たり判定との分離

`FieldDefinition.objects`と既存の衝突矩形を唯一の正とする。本番素材は、オブジェクト座標・幅・高さ・`solid`・出口範囲・木座標・NPC座標・主人公半径・移動速度を変更してはならない。

画像が衝突矩形より大きく張り出す場合も、歩ける部分と通れない部分が見た目で矛盾しないよう透明余白とアンカーを調整する。柵、水路、田んぼ、藪、竹林は単一画像を引き伸ばさず、`repeat-x`、`repeat-y`、短いタイルを使う。

装飾はゲームセーブと乱数へ影響させず、道や出口を塞いで見せず、木の気配と誤認する虫影を置かない。画面外の装飾アニメーションは停止する。

## 10. キャラクターと虫

### 10.1 主人公

~~~ts
interface CharacterSpriteSheetDefinition {
  assetId: VisualAssetId;
  frameWidth: number;
  frameHeight: number;
  anchor: { x: number; y: number };
  frames: Record<
    FacingDirection,
    { idle: number; walk: [number, number, ...number[]] }
  >;
}
~~~

- 8方向移動中も既存`FacingDirection`の4方向表示を使う。
- 虫取り網、帽子、虫かごを小さい画面で判別できる。
- 歩行フレームは現在の移動速度と同期し、停止時は待機へ戻る。
- 動きを減らす設定では歩行フレームを切り替えず、位置移動だけを行う。

NPCは配色、服、体格、持ち物で見分けられる静止画を最低要件とする。任意の呼吸・瞬きは動きを減らす設定で停止する。未発見の人物図鑑へ本番画像を漏らさない。

### 10.2 接写背景と虫

| `InspectionSceneKind` | 表現 |
| --- | --- |
| `sap` | 樹液のにじみ、樹皮の凹凸 |
| `bark-crack` | 幹の割れ目、うろ、暗い奥行き |
| `root` | 根、土、落ち葉 |
| `banana-trap` | 木へ結んだ餌、樹皮 |
| `light-trap` | 灯り、受け面、夜の背景 |

背景は虫を含まない状態で作り、捕獲対象と環境昆虫を必ず別素材にする。捕獲対象5種は角、大顎、体形、色で判別でき、脚や角の接続に明確な破綻がないこと。画像の大きさは`sizeMm`と直接比例させず、タップ領域の有利不利を生まない。

環境昆虫は既存の`motion`で`still`、`crawl`、`flutter`を表現し、操作対象にせず`aria-hidden="true"`を維持する。動きを減らす設定では静止表示にする。画像側で数、種類、位置を増減せず、既存セッションデータを使う。

## 11. Audio Asset Manifest

### 11.1 型定義

~~~ts
type AudioBus = "ambient" | "effects";

type AudioLoadGroup =
  | "title"
  | "current-field"
  | "neighbor-field"
  | "inspection"
  | "event"
  | "lazy";

interface AudioSource {
  src: string;
  mimeType: "audio/ogg" | "audio/mp4";
}

interface AudioAssetDefinition {
  id: string;
  bus: AudioBus;
  sources: AudioSource[];
  loop: boolean;
  loadGroup: AudioLoadGroup;
  defaultGain: number;
  loopStartSeconds?: number;
  loopEndSeconds?: number;
  licenseId: string;
}
~~~

Opus系とAAC系を用意し、`canPlayType`で対応形式を選ぶ。

### 11.2 環境音

最低限、次の再利用可能なレイヤーを用意する。

- 朝の鳥、昼の蝉、夕方のヒグラシ、夜のコオロギ
- 田んぼ道の水、竹林の風、林の葉音
- 裏庭夜の虫、秘密の森の弱い風
- 接写時の小さな羽音、ライトトラップ周辺の低い環境音

`FieldTheme`と`TimePeriod`から基礎レイヤーと場所固有レイヤーを合成し、全組み合わせへ定義または明示的な標準フォールバックを持つ。

### 11.3 効果音

- UIを開く／閉じる
- 土・草・石の歩行
- フィールド切替
- 木の接写開始、小さな羽音
- 網を振る、捕獲成功
- トラップ設置、回収
- 18時のお迎え、一日のまとめ

環境音は20〜45秒程度の継ぎ目が目立たないループ、効果音は原則1.5秒以内とする。True Peakは-1 dBTP以下を目安とし、環境音は効果音より明確に小さくする。

## 12. 音声アーキテクチャ

~~~text
AudioContext
  └─ masterGain
      ├─ ambientGain
      │   ├─ activeAmbientA
      │   └─ activeAmbientB
      └─ effectsGain
~~~

- 2組の環境音スロットを交互に使い、400〜800msでクロスフェードする。
- ミュート時はMaster Gainを滑らかに0へ下げる。
- AudioContextは1個だけ作成し、React再描画で増やさない。
- `visibilitychange`で非表示になったら停止し、復帰時は最新Sceneだけを再開する。
- 高速な場面遷移では古い読込・再生要求をキャンセルする。
- 効果音へEvent Tokenを渡し、同じゲームイベントを1回だけ再生する。
- 保存済み`pendingOutcome`の復元、React Strict Mode、再描画で捕獲音を再生しない。
- 音声用の揺らぎにゲームの決定的乱数を使わない。

## 13. Preferences Schema Version 1

~~~ts
interface UserPreferencesV1 {
  schemaVersion: 1;
  soundEnabled: boolean;
  ambientVolume: number;
  effectsVolume: number;
  motion: "system" | "reduced";
}

const defaultPreferences: UserPreferencesV1 = {
  schemaVersion: 1,
  soundEnabled: false,
  ambientVolume: 0.55,
  effectsVolume: 0.8,
  motion: "system",
};

export const PREFERENCES_KEY = "kabukuwa.preferences.current";
~~~

- Zodで構造検証し、音量は0〜1へ丸める。
- 不正な値は項目単位で既定値へ戻す。
- 壊れた設定のためにゲームセーブを破棄しない。
- LocalStorageが利用できない場合はメモリ内設定で継続する。
- Preferencesへバックアップ世代は設けない。
- `RESET_GAME`でPreferencesを削除しない。
- 「設定を初期値へ戻す」だけがPreferencesを初期化する。
- 設定変更で`GameState.revision`を増やさず、ゲーム内時間も進めない。

本機能では`GameState.schemaVersion`と`contentVersion`を6のまま維持し、`kabukuwa.save.pre-v7`を作らない。Asset Manifestは`assetVersion: 1`としてゲームセーブとは別に管理する。

## 14. 既存機能との統合

### 14.1 探索・接写・日替わり

- 移動速度、当たり判定、カメラ、出口判定を変更しない。
- 「！」、トラップ、調査済みマーカーの意味を変更しない。
- 日次Natureに応じて表示上の環境音量や装飾密度を変えてもよいが、出現補正を再計算しない。
- 過去の観察ノートを開いた時に、その日の環境音を自動再生しない。
- Waiting、Ready、Openedを絵と文章で判別できるようにするが、Ready表示で虫の有無を示さない。

### 14.2 とっておきの虫かご

- 虫画像を個体ごとに複製保存せず、`Specimen.insectId`から解決する。
- とっておき設定・解除には短いUI音だけを使用できるが、捕獲音を使わない。
- 500個体表示時も同じ画像URLを共有する。

### 14.3 18時・20時

- 17:55の接写は閉じるまで同じ場面・音を維持する。
- 接写を閉じた後、迎え音と夕方イベントへ1回だけ切り替える。
- 20時直前の接写も、結果を閉じるまで接写音を維持する。
- 一日のまとめを再描画しても終了音を繰り返さない。
- 翌日開始時は朝環境音へ1回だけ切り替える。

## 15. 素材の権利と管理

原則として、本作向けオリジナル素材、本作向けに生成して利用条件を満たす素材、または商用利用・改変・再配布条件を確認済みの素材だけを使う。

次を禁止する。

- 検索結果から出典不明の画像や音を保存する。
- 外部サイトのURLを本番画面から直接読み込む。
- 既存ゲーム、映画、アニメのキャラクターや背景を模倣する。
- ライセンス確認前の素材を仮のまま本番へ残す。
- クレジット必須素材の表示義務を省略する。

`docs/art/asset-provenance.md`へ、Asset ID、ファイル名、作成方法または出典、作成日、ライセンス、加工内容、クレジット要否、確認者、備考を記録する。Manifestの全`licenseId`がProvenance内に存在することをビルド検査する。

## 16. アクセシビリティ

- 画像内へ操作に必要な文字を埋め込まない。
- 装飾画像は`alt=""`または`aria-hidden="true"`とする。
- 場所名、NPC名、虫名、時間、結果は既存HTMLで伝える。
- 捕獲前の操作名で虫種を読み上げず、捕獲後に種類とサイズを伝える。
- 音だけを気配、Ready、捕獲成功、時間帯の唯一の伝達手段にしない。
- 音のオン／オフ状態を`aria-pressed`で公開する。
- 音量スライダーへ現在値と用途を読み上げるラベルを付ける。
- 設定、ミュート、操作説明のタップ領域を44×44 CSS px以上にする。
- Escape、ブラウザ戻る、画面内の戻るを同じ階層動作にする。
- 設定を閉じた後は元の設定ボタンへフォーカスを戻す。
- 200%拡大時も横スクロールなしで操作できる。
- `prefers-reduced-motion: reduce`では手動設定に関係なく過度な動きを停止する。
- 光の点滅は1秒3回を超えない。
- 夜画面と`forced-colors`でもボタン、文字、フォーカスを識別できる。
- 音声開始失敗をモーダルエラーにせず、ゲームを継続する。

## 17. エラー・境界ケース

### 17.1 画像

- 画像404では該当レイヤーだけCSSへ戻す。
- WebP失敗時はPNGを試し、1枚の失敗で場面全体を隠さない。
- 高速なフィールド往復で古い読込完了が現在画面へ混入しない。
- 接写を閉じた後に完了した画像Promiseを破棄する。
- 壊れたURLを無限再試行しない。
- 素材未定義時は開発環境で明示エラー、本番でCSSフォールバックにする。

### 17.2 音声と設定

- 対応Codecがない場合は無音で継続する。
- AudioContext開始拒否でUnhandled Promise Rejectionを出さない。
- ミュート連打や連続出口移動でContext・ループを増やさない。
- 捕獲結果でとっておきを切り替えても捕獲音を再生しない。
- `pendingOutcome`を保持した再読込で捕獲音を再生しない。
- JSON破損、未知Version、保存容量エラーでもゲームを継続する。
- ゲームの「はじめから」で音設定を失わず、設定初期化でゲームセーブを削除しない。

### 17.3 ゲーム境界

最低限、17:55の通常・トラップ接写、19:50以降の裏庭接写、捕獲結果からのお気に入り設定、18時・20時の保留、Readyを数日放置したトラップ、夜に再開した旧セーブ、素材読込中のブラウザ戻る、設定中の画面回転を確認する。

## 18. 性能要件

- タイトル必須画像: 合計700KB以内を目安。
- 最初のフィールド本番素材: 合計1.2MB以内を目安。
- 最初の環境音: 合計500KB以内を目安。
- 初回操作可能までの総追加転送: 2.5MB以内を目安。
- 単一実行素材: 原則1MB未満。
- 最適化済み素材全体: 25MB以内を目安。
- JavaScript増加: 素材URLを除き40KB gzip以内を目安。
- 起動時に全11フィールドを読み込まない。
- 現在地と直接接続する隣接先だけを先読みする。
- 接写素材は木へ近づいた時点か接写直前、図鑑素材は図鑑を開くまで読み込まない。
- 通常時60fpsを目標とし、45fps未満が継続しない。
- 画像表示開始によるLayout Shiftを発生させない。
- 360×640でフィールドUIと設定ボタンを重ねず、Safe Areaを考慮する。

## 20. テスト仕様

### 20.1 素材・設定・音声の単体テスト

- 全`FieldId`、`FieldObjectKind`、`InspectionSceneKind`、`InsectId`、`AmbientInsectId`を網羅する。
- 全素材ファイルと`licenseId`が存在し、外部HTTP URLを含まない。

- 設定初期化と`RESET_GAME`が互いの保存領域を削除しない。
- 全`FieldTheme × TimePeriod`に環境音Sceneまたは標準フォールバックがある。

アート・音声実装からゲームの決定的乱数ヘルパーを呼び出していないことも確認する。

### 20.3 結合テスト

- タイトルで音をオンにし、開始後に現在地の環境音へ移る。
- 音オフではAudioContextを不要に起動しない。
- 家→田んぼ道→神社で古い環境音が残らない。

- 接写背景と虫が正規化座標へ表示され、捕獲音が1回だけ鳴る。
- 再読込やとっておき変更で捕獲音が鳴らない。
- トラップを昼・夕方に開いて朝固定の表現にならない。
- 17:55・19:50の接写が閉じるまで元の時間帯を維持する。
- 設定中は歩行が止まり、閉じると元のボタンへフォーカスが戻る。
- 操作説明再表示でセーブRevisionが変わらない。

### 20.4 画面・音声確認

## 21. 完了条件

次をすべて満たした時点で「夏の村・本番アートと環境音 Ver.6」を完成とする。
