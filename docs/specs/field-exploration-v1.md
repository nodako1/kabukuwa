# カブクワ 次期機能仕様書

## 歩いて探すフィールド探索 Ver.1

- 作成日: 2026-07-21
- 対象: `nodako1/kabukuwa`
- 対象環境: スマートフォン向けWebアプリ（Vercel本番環境）
- ステータス: 実装候補仕様

## 1. 結論

次に実装する機能は、現在の「地図から行き先ボタンを押して移動する方式」を、主人公を実際に歩かせる見下ろし型2Dフィールド探索へ進化させる機能とする。

完全な一枚のオープンワールドにはせず、各場所と道中を複数のフィールドに分ける。フィールド内では主人公を上下左右へ動かし、画面は主人公を追いかけて縦横にスクロールする。フィールドの端にある出口を通ると、次の道や場所へ移動する。

既存の虫抽選、NPC、時間、18時の迎え、夜の裏庭、広告公平性、図鑑、セーブは維持する。

## 2. 目的

### 2.1 ゲーム体験上の目的

- 「場所を選ぶ」だけでなく、「自分で歩いて虫を探しに行く」体験を作る。
- おばあちゃんの家から林までの距離や、田舎の道中そのものを夏休みの思い出にする。
- 木、NPC、秘密の道を自分で見つける楽しさを追加する。
- 固定された地形を覚える攻略性と、虫がいる木だけが変わるランダム性を両立する。

### 2.2 開発上の目的

- 今後の捕獲ミニゲーム、落とし物、天候、道中イベント、足音・環境音を追加できる土台を作る。
- 現在のReact製ゲームロジックを壊さず、表示・入力部分を段階的に強化する。

## 3. 採用するゲーム形式

### 3.1 表現

- 視点: 少し斜め上から見下ろす2Dトップダウン方式
- 画面: スマートフォン縦持ちを基準とする
- フィールド: 画面より縦横に広い固定マップ
- カメラ: 主人公を追従し、フィールド端では停止する
- 移動方向: 上下左右と斜めを含む8方向
- マップ構成: 場所ごとに区切られた複数シーン方式

### 3.2 採用しない方式

- 一枚の巨大な完全オープンワールド
- 横スクロールアクション
- 3D表示
- 歩行中に現実時間と同じ速度でゲーム内時計が進む方式
- 木やNPCを遠くからタップするだけで即座に行動できる方式

## 4. ワールド構成

既存の8場所に、道中用フィールドを3つ追加する。

### 4.1 採集・交流フィールド

1. おばあちゃんの家
2. 裏庭
3. 神社
4. 雑木林
5. クヌギ林
6. 竹林
7. 小学校
8. 秘密の森

### 4.2 道中フィールド

1. 田んぼ道
   - おばあちゃんの家と神社をつなぐ
   - 用水路、田んぼ、電柱、道しるべを配置する
2. 林道
   - 神社、雑木林、クヌギ林、竹林、小学校をつなぐ
   - 分かれ道によって行き先を選べる
3. 秘密の小道
   - 神社と秘密の森をつなぐ
   - 条件未達時は草木で塞がれている
   - 条件達成後も16:00〜18:00だけ通れる

### 4.3 接続イメージ

```text
裏庭
  ↕
おばあちゃんの家
  ↕
田んぼ道
  ↕
神社 ─ 秘密の小道 ─ 秘密の森
  ↕
林道
  ├─ 雑木林
  ├─ クヌギ林
  ├─ 竹林
  └─ 小学校
```

実際の接続は迷いすぎない範囲で調整する。すべての出口には、行き先が分かる看板や景色の目印を置く。

## 5. 主人公の操作

### 5.1 スマートフォン

- 画面左下に半透明の仮想スティックを表示する。
- 指を倒した方向へ主人公が歩く。
- 斜め入力を含む8方向移動に対応する。
- 画面右下に状況対応型の「行動」ボタンを表示する。
- スティック操作中はブラウザ画面そのものがスクロールしない。
- 図鑑や人物図鑑などのシート内は従来どおり縦スクロールできる。

### 5.2 PC

- 矢印キーまたはWASDで移動する。
- EnterまたはSpaceで行動する。
- ボタン操作も併用できる。

### 5.3 移動感

- 入力から移動開始までの遅延を感じさせない。
- 通常歩行のみ実装し、ダッシュはVer.1対象外とする。
- 主人公が停止したときは向いている方向を保持する。
- 移動中は最低2枚、目標3枚の歩行アニメーションを表示する。

## 6. カメラとスクロール

- 主人公が画面中央付近に来るようカメラを追従させる。
- フィールドが横に広い場所では横スクロールする。
- 縦に長い道では縦スクロールする。
- フィールド端では背景の外側を表示しない。
- モーダル、虫発見結果、会話、18時イベント中はカメラと移動を停止する。
- `prefers-reduced-motion` が有効な場合はカメラ追従の補間を弱めるか無効化する。

## 7. 地形と当たり判定

### 7.1 通れないもの

- 建物
- 大木の幹
- 石垣
- 柵
- 用水路
- 深い藪
- マップ外周

### 7.2 通れるもの

- 道
- 草地
- 神社の境内
- 校庭
- 木の根元に近づくための空間

### 7.3 表示順

- 主人公より画面下側にある木やNPCは主人公の手前に表示する。
- 主人公より画面上側にあるものは主人公の奥に表示する。
- 木の葉の下を歩いたときに一部が隠れ、森の中を歩いている感覚を出す。

## 8. 木・トラップの探索

### 8.1 基本動作

1. 主人公が木やトラップへ近づく。
2. 右下の行動ボタンが「クヌギを覗く・15分」に変わる。
3. ボタンを押す。
4. 現在と同じ虫抽選を行う。
5. 「いる」または「いない」の結果を表示する。

### 8.2 探索ルール

- 木やトラップの位置は毎回固定する。
- 虫がいる対象だけを訪問ごと・時間帯ごとに変える。
- 調査済みの木には控えめな印を表示する。
- 遠くの木は直接選択できない。
- 複数の対象が近い場合は、主人公に最も近いものを行動対象にする。
- 虫の大きさ抽選と広告効果の分離は維持する。

## 9. NPCとの会話

- NPCは現在と同じ時間帯・日付条件で出現する。
- 主人公がNPCへ近づくと、行動ボタンが「話す」に変わる。
- 会話中は主人公とNPCを停止する。
- 話しかけた回数と段階会話を維持する。
- NPCの立ち位置は時間帯ごとに固定し、「今日はどこにいるか」を探せるようにする。
- 神社のおじさん3回目の会話後、秘密の小道の入口が視覚的に変化する。

## 10. 場所の移動とゲーム内時間

### 10.1 時間の進み方

- 同じフィールド内を歩くだけではゲーム内時間を進めない。
- 木の探索とNPC会話は従来どおり15分進める。
- 出口から隣のフィールドへ移ると、接続ごとに設定された5〜10分を進める。
- 休憩は従来どおり30分進める。

この方式により、操作が苦手なプレイヤーだけ時間を失う問題を防ぐ。

### 10.2 出口

- 出口に近づくと「神社へ進む・5分」などと表示する。
- 行動ボタンを押すと次のフィールドへ移る。
- 誤操作防止のため、出口へ触れただけでは移動しない。
- 到着時は出口の反対側にある安全な地点へ主人公を配置する。

### 10.3 18時の迎え

- 遠方で時間が18:00に到達した場合は、その場で操作を止める。
- 「おーい！迎えに来たよ！」のイベントを表示する。
- 暗転後、おばあちゃんの家の玄関前へ移動する。
- 時刻は18:15、行動可能範囲は家と裏庭だけにする。
- 18時以降は遠方へ続く出口を閉じ、理由を表示する。

## 11. 地図画面

- 現在の行き先ボタンによる瞬間移動は廃止する。
- 地図は「現在地と道を確認する画面」へ変更する。
- 現在地、発見済みの場所、未解放の場所、通れない時間帯を表示する。
- 秘密の森は解放前には地図へ表示しない。
- 最初から近道・ファストトラベルは実装しない。
- 移動が繰り返し作業になる場合は、今後「一度歩いた道を省略する」機能を検討する。

## 12. セーブと再開

### 12.1 保存対象

- 現在のフィールドID
- 主人公のX・Y座標
- 主人公の向き
- 最後に安全だった座標
- 発見済みフィールド
- 調査済みの木
- 従来の時刻、図鑑、NPC会話、秘密ルート、広告効果など

### 12.2 保存タイミング

- 木やNPCに対する行動後
- 別フィールドへ移動した直後
- 主人公が停止してから一定時間後
- ブラウザがバックグラウンドへ移るとき

### 12.3 既存セーブ移行

- セーブ形式をSchema Version 2へ更新する。
- Version 1のセーブを読み込んだ場合は、保存されている場所に対応する標準開始地点へ配置する。
- 図鑑、捕獲個体、NPC会話回数などは失わない。
- 読み込みに失敗した場合はバックアップを試し、ゲームを停止させない。

## 13. 画面UI

### 13.1 維持するもの

- 上部の時刻・日付・現在地
- 図鑑
- 人物図鑑
- 休憩
- 広告報酬
- 虫発見結果
- 18時イベント
- 一日のまとめ

### 13.2 変更するもの

- 現在の木を直接タップする画面を、主人公が歩くフィールドへ置き換える。
- 下部の固定アクションカードを、近くの対象に応じて変わる行動ボタンへ置き換える。
- 地図ボタンは移動画面ではなく確認画面にする。
- 操作説明は初回だけ短く表示し、その後は設定から再確認できるようにする。

## 14. アート要件

### 14.1 主人公

- 夏休みの小学生
- 虫かご、帽子、虫取り網を装備
- 上下左右4方向の待機絵
- 上下左右4方向、各2〜3枚の歩行絵
- 小さい画面でも主人公の向きが分かる配色

### 14.2 背景

- 昭和から平成初期を思わせる、懐かしい日本の田舎
- 木、道、建物の形で場所を見分けられる
- 朝、昼、夕方、夜で色味を変える
- 移動できる道と通れない地形が直感的に分かる
- 木の見た目だけで虫の有無を判別できないようにする

### 14.3 初期実装

- 移動システム完成を優先し、最初はコードベースの簡易背景と仮キャラクターでもよい。
- 操作と当たり判定が確定した後、統一された画像素材へ差し替える。

## 15. 技術設計方針

### 15.1 描画

- 現在のReact構成を維持する。
- フィールド、主人公、木、NPCをワールド座標で絶対配置する。
- カメラ移動はフィールド全体へのCSS Transformで表現する。
- 移動更新は`requestAnimationFrame`を使用する。
- Ver.1では新しい大型ゲームエンジンを導入しない。

### 15.2 データ

各フィールドをデータ定義し、表示コードへ座標を直接書き込まない。

```ts
interface FieldDefinition {
  id: FieldId;
  locationId: LocationId;
  width: number;
  height: number;
  spawnPoints: SpawnPoint[];
  exits: FieldExit[];
  obstacles: CollisionShape[];
  hotspots: PositionedHotspot[];
  npcPositions: NpcPosition[];
}
```

ゲーム状態へ次を追加する。

```ts
interface PlayerFieldState {
  fieldId: FieldId;
  x: number;
  y: number;
  facing: "up" | "down" | "left" | "right";
  lastSafeX: number;
  lastSafeY: number;
  discoveredFieldIds: FieldId[];
}
```

### 15.3 性能

- 通常時は60fpsを目標とし、最低45fpsを下回り続けない。
- 画面外の複雑なアニメーションを停止する。
- フィールド画像は必要になる直前に読み込む。
- 起動時に全マップの大型画像を一括読み込みしない。
- 同一フレームでReact Stateを大量更新せず、移動表示と永続ゲーム状態を分離する。

## 16. 実装順

### Step 1: 移動エンジン

- 主人公表示
- 仮想スティック、キーボード入力
- 8方向移動
- カメラ追従
- フィールド端制御
- 当たり判定

### Step 2: 縦切り動作確認

- おばあちゃんの家
- 田んぼ道
- 神社
- 木を1本調べる
- NPCと1回話す
- 場所移動で時間が進む

この段階では現在の地図移動を非常用の代替手段として残す。

### Step 3: 全フィールド展開

- 既存8場所へ展開
- 林道と秘密の小道を追加
- 全出口と時間制限を接続
- 地図の瞬間移動を廃止

### Step 4: セーブ移行と仕上げ

- セーブVersion 2
- 同じ位置から再開
- 18時イベント
- 夜の裏庭
- 操作説明
- モバイル性能調整
- 非常用の位置リセット

各Stepは既存の遊びを壊さない状態で統合し、最終的にVer.1を有効化する。

## 17. テスト仕様

### 17.1 単体テスト

- 入力方向から移動量への変換
- 斜め移動の速度補正
- カメラ位置の上限・下限
- 矩形・円形の当たり判定
- 最も近い行動対象の選択
- フィールド接続と移動時間
- 18時到達時の迎え判定
- セーブVersion 1からVersion 2への移行

### 17.2 結合テスト

- 家から神社まで実際に歩いて移動できる
- 木へ近づかないと探索できない
- 木を調べると15分進む
- NPCへ近づくと会話できる
- 秘密の小道が条件と時間帯の両方で開閉する
- 18時に遠方から家へ戻される
- 18時以降は裏庭以外へ進めない
- 再読み込み後も同じ場所・ほぼ同じ位置から再開できる

### 17.3 画面確認

- iPhone相当 360×640
- iPhone相当 390×844
- 大型スマートフォン 430×932
- PCキーボード操作
- 縦横スクロール時にブラウザ余白が見えない
- モーダル表示中に主人公が動かない

## 18. 完了条件

次をすべて満たした時点で「歩いて探すフィールド探索 Ver.1」を完成とする。

1. 主人公をスマートフォンとPCの両方で8方向へ動かせる。
2. 主人公に追従して画面が縦横へスクロールする。
3. 建物、木、水路などを通り抜けない。
4. 既存8場所と3つの道中フィールドを歩いて移動できる。
5. 木、トラップ、NPCへ近づいて行動できる。
6. 虫抽選、広告公平性、図鑑記録が現在と同じように動く。
7. 秘密ルートと18時以降の制限が正しく動く。
8. 途中終了後、同じフィールド・同じ位置から再開できる。
9. 既存セーブデータを失わない。
10. 主要スマートフォンサイズで操作不能や横方向の画面崩れがない。
11. 自動テスト、型チェック、本番ビルドが成功する。

## 19. Ver.1の対象外

- 捕獲アクションゲーム
- ダッシュ、スタミナ
- 自転車などの乗り物
- 道中でのランダム虫遭遇
- 雨、台風などの天候変化
- オンラインランキング
- マルチプレイ
- クラウドセーブ
- 完成版の全背景・全キャラクターイラスト

## 20. 次に検討する機能

フィールド探索Ver.1の完成後は、次の順で検討する。

1. 虫を見つけた後の捕獲ミニゲーム
2. 足音、蝉、風、川などの環境音
3. 道中の小さなランダムイベント
4. 一度歩いた道を省略できる近道
5. 統一された完成版イラストへの差し替え
